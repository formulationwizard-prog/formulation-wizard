# CFR Watcher — Regulatory Currency Monitoring

**Status:** MVP shipped 2026-05-24. Foundation for ongoing regulatory currency discipline.

**Origin:** Operator + Opus + CC session 2026-05-24. Built same day as the FDA Nutrition Facts rounding compliance fix (b0919ec) which surfaced the underlying risk: silent regulatory drift between FDA rule changes and codebase encoding produces non-compliant labels customers can't detect without an audit.

## Problem

The platform encodes regulatory rules across multiple CFR parts (101.9, 101.36, 111, 113/114, FALCPA, FASTER Act, DSHEA). When FDA amends these regulations:

1. The amendment is published in the Federal Register
2. eCFR is updated within days
3. Manufacturers using outdated label templates continue shipping non-compliant products
4. Enforcement catches up after recalls, state AG actions, or retail QA audits
5. Class-action liability and reputational damage compound

**The 2016 NFP overhaul** that made Vit D / Calcium / Iron / Potassium mandatory declarations is the textbook example — manufacturers using pre-2016 templates shipped non-compliant labels for months before remediation.

Pre-launch this risk is bounded (no customers yet). Post-launch with paying customers shipping labels generated by the platform, the risk is direct legal liability + platform reputational damage.

## Solution shape

Automated weekly polling of regulatory sources, structured diffing against stored baselines, GitHub-issue surfacing with triage tags for human review.

**This is not unattended remediation.** The watcher detects + classifies + surfaces; humans (operator + Opus + PA) decide what to change in the codebase. Same honest-estimate + PA-as-authority discipline the platform applies to nutrition values, applied to regulations themselves.

## Architecture

```
cfr-watch-list.json     ← Manifest: what to watch + why
scripts/cfr-watch/
  cfr-watch.mjs         ← Entry script + main loop
  fetcher.mjs           ← Pluggable RegSourceFetcher interface
  fetchers/ecfr.mjs     ← eCFR Versioner API adapter (US federal)
  fetchers/govinfo.mjs  ← govinfo Public Laws adapter
  differ.mjs            ← Hash + line-diff + heuristic classifier
  reporter.mjs          ← Console + GitHub-issue body formatters
data/cfr-baselines/     ← Stored content + hash per watched section
.github/workflows/cfr-watch.yml  ← Weekly cron + issue opener
```

## Pluggable fetcher architecture

`RegSourceFetcher` interface enables extension to additional regulatory sources without modifying the main loop. Current adapters:

- **eCFR** (federal CFR sections) — uses Versioner API; resolves correct effective date via `/titles` lookup
- **govinfo** (Public Laws) — fetches Public Law HTML pages

Future adapter targets per Opus routing 2026-05-25 (Q2: design extensible, ship US-only data):

- **CA Prop 65** — California Office of Environmental Health Hazard Assessment portal
- **NY allergen labeling** — state-specific updates
- **EUR-Lex** — EU regulations (1169/2011 for FIC, etc.)
- **CFIA** — Canadian Food Inspection Agency
- **TGA** (Therapeutic Goods Administration) — Australia supplement regulations
- **HSA** — Singapore supplement regulations

Each is a single new file `scripts/cfr-watch/fetchers/<source>.mjs` + a registration line in `fetcher.mjs` + corresponding manifest entries. No changes to the main loop or reporter required.

## Triage tag scheme

Per Opus routing 2026-05-25 (Q3: tiered routing). Heuristic classifier in `differ.mjs classifyChange()` assigns a first-pass tag; humans re-tag on review.

| Tag | Meaning | Routes to |
|---|---|---|
| `code-impact` | Encoded business logic affected (rounding, allergen lists, thresholds, mandatory declarations) | Opus architecture session → CC implementation |
| `interpretation-impact` | Compliance interpretation affected (disclaimer language, "significant amount" thresholds, guidance updates) | PA (when retained) → operator + Opus pre-launch |
| `no-impact` | Formatting / punctuation / non-substantive editorial | Close as informational; re-baseline entry |
| `needs-discussion` | Ambiguous — could be any of above | Operator + Opus session |

Classifier is conservative: when uncertain, tags `needs-discussion` rather than `no-impact`. Better to over-surface than silently dismiss.

## Operational commands

```bash
# Run watcher locally (console output)
npm run cfr:watch

# Run watcher in CI mode (GitHub issue body output)
npm run cfr:watch -- --github

# Refresh all baselines (after reviewing a confirmed change and applying any code updates)
npm run cfr:baseline

# Refresh single baseline
npm run cfr:baseline -- <entry-id>
```

## Exit codes

| Code | Meaning | CI action |
|---|---|---|
| 0 | All unchanged or first-baseline | Success; no issue created |
| 1 | Fetch errors (transient or permanent) | Workflow fails; investigate logs |
| 2 | Substantive changes detected | Workflow opens GitHub issues; operator triage |

## Review cadence

- **Weekly:** Automated polling on Sunday 06:00 UTC
- **Per issue:** Operator triages within ~1 week of issue open, escalates per tag
- **Quarterly:** Operator + Opus review watch list itself — confirm entries still cover the codebase's regulatory surface, identify gaps, retire deprecated entries

## Limitations + future work

**Phase 1 (current MVP):**
- Hash-based change detection on whole-section content
- Heuristic classifier (keyword corpus only)
- Public-law adapter watches static law text, not amendment activity
- US federal scope only

**Phase 2 candidates** (post-launch):
- **Federal Register adapter** — watch for new rules referencing existing watched CFR sections (catches amendments before they propagate to eCFR)
- **Sub-section granularity** — watch (c) or (d)(7) specifically, not whole section, reducing diff noise
- **Citation manifest integration** — every platform rule references its CFR citation in code; CC sweep flags un-cited rules; ties watcher findings to specific codebase locations automatically
- **NLP-based classifier** — replace keyword heuristic with embedding-based change classification
- **State + international adapters** — Prop 65, EU 1169/2011, etc.
